

# خطة: إضافة حقل قيمة السيارة عند اختيار شامل

## المشكلة

عند إنشاء وثيقة جديدة واختيار نوع "شامل" (FULL)، إذا كانت السيارة المختارة لا تحتوي على قيمة (`car_value`)، لا يمكن حساب سعر التأمين بشكل صحيح لأن حساب الشامل يعتمد على:
```
fullComponent = carValue * (fullPercent / 100)
```

## الحل

### 1. إضافة حقل قيمة السيارة في Step3PolicyDetails

عند اختيار "شامل" كنوع فرعي، يظهر حقل جديد لقيمة السيارة:
- **إذا كانت السيارة لديها قيمة**: يُعرض الحقل مع القيمة المخزنة ويمكن للمستخدم تعديلها
- **إذا لم تكن لديها قيمة**: يُحاول جلب القيمة من واجهة الحكومة، وإذا لم تُوجد يُطلب من المستخدم إدخالها

```text
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ يظهر حقل "قيمة السيارة" تلقائياً                        │
│                                                          │
│ • إذا السيارة لديها car_value → يُعرض في الحقل          │

│ عند الحفظ: يُحدث car_value في جدول السيارات             │
└─────────────────────────────────────────────────────────┘
```

### 5. منطق الجلب التلقائي

```typescript
const fetchCarValue = async () => {
  const car = selectedCar || existingCar;
  if (!car?.manufacturer_name || !car?.year) return;
  
  setFetchingCarValue(true);
  const { data } = await supabase.functions.invoke('fetch-car-price', {
    body: {
      manufacturer: car.manufacturer_name,
      model: car.model,
      year: car.year
    }
  });
  
  if (data?.found && data?.data?.price) {
    setPolicy({ ...policy, full_car_value: data.data.price.toString() });
  }
  setFetchingCarValue(false);
};
```

### 6. التحقق (Validation)

```typescript
// في validateStep للخطوة 3
if (policy.policy_type_child === 'FULL') {
  const carValue = policy.full_car_value || 
                   selectedCar?.car_value?.toString() || 
                   newCar.car_value;
  if (!carValue || parseFloat(carValue) <= 0) {
    newErrors.full_car_value = "قيمة السيارة مطلوبة للتأمين الشامل";
  }
}
```

### 7. عند الحفظ - تحديث السيارة

```typescript
// في handleSave
if (policy.full_car_value && carId) {
  await supabase
    .from('cars')
    .update({ car_value: parseFloat(policy.full_car_value) })
    .eq('id', carId);
}
```

## النتيجة المتوقعة

- عند اختيار "شامل"، يظهر حقل قيمة السيارة تلقائياً
- إذا كانت السيارة لديها قيمة مخزنة، تُعرض في الحقل
- إذا لم تكن، يُحاول النظام جلبها من API الحكومة
- المستخدم يستطيع دائماً تعديل القيمة يدوياً
- لا يمكن المتابعة بدون قيمة سيارة صالحة
- القيمة تُحفظ في جدول السيارات عند إتمام الوثيقة
