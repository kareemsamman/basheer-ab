
# خطة: إعادة تصميم عرض الباقة كوحدة موحدة

## فهم المشكلة الحالية

### الوضع الحالي:
1. **في صفحة العميل**: الباقة تُعرض كبطاقة واحدة مع badge "باقة" ✅
2. **في تفاصيل الوثيقة (Drawer)**: عند الضغط على الوثيقة، كل وثيقة تُفتح بشكل منفصل
3. **مشكلة الدفعات**: الدفعات موزعة على الوثائق بشكل منفصل:
   - وثيقة ثالث/شامل تظهر 0 دفعات
   - وثيقة إلزامي تظهر 2 دفعات (₪2,000)
4. **مشكلة الملفات**: كل وثيقة لها ملفاتها الخاصة

### المشكلة:
الباقة ليست **وحدة موحدة** - إنما هي مجموعة وثائق منفصلة مربوطة بـ `group_id`

---

## الحل المقترح: واجهة باقة موحدة

### المفهوم الجديد:
- الباقة = وثيقة رئيسية واحدة تحتوي على عدة أنواع تأمين
- عند فتح التفاصيل → يُعرض **ملخص الباقة** بدلاً من وثيقة واحدة
- الدفعات والملفات تكون للباقة كاملة وليس لكل وثيقة

---

## التغييرات المطلوبة

### 1. تعديل `PolicyDetailsDrawer` لعرض الباقة

#### العنوان الرئيسي:
```
باقة تأمين
━━━━━━━━━━━━━━━━━━━━━━━━
العميل: Kareem Test | السيارة: 12345678 | الشركة: أراضي مقدسة
```

#### جدول مكونات الباقة (جديد):
| نوع التأمين | الفترة | السعر |
|-------------|--------|-------|
| ثالث/شامل - طرف ثالث | 29/01/2026 → 28/01/2027 | ₪1,000 |
| إلزامي | 29/01/2026 → 28/01/2027 | ₪1,000 |
| **المجموع** | | **₪2,000** |

#### بطاقات مالية موحدة:
- سعر التأمين: ₪2,000 (مجموع الباقة)
- المدفوع: ₪2,000 (مجموع كل الدفعات)
- المتبقي: ₪0

### 2. الدفعات موحدة
- تاب "الدفعات" يعرض **كل دفعات الباقة** من جميع الوثائق
- عند إضافة دفعة جديدة → يتم توزيعها على وثائق الباقة تلقائياً

### 3. الملفات موحدة
- تاب "الملفات" يعرض **كل ملفات الباقة** من جميع الوثائق
- عند رفع ملف → يمكن اختيار لأي وثيقة في الباقة

## التنفيذ التقني

### الملفات المتأثرة:

| الملف | التغيير |
|-------|---------|
| `PolicyDetailsDrawer.tsx` | تحويل العرض من وثيقة واحدة إلى عرض باقة موحد |
| `PolicyPaymentsSection.tsx` | جلب دفعات كل وثائق الباقة وعرضها معاً |
| `PolicyFilesSection.tsx` | جلب ملفات كل وثائق الباقة وعرضها معاً |

### منطق الجلب الجديد:

```typescript
// عند فتح وثيقة لها group_id
if (policy.group_id) {
  // جلب كل وثائق الباقة
  const { data: packagePolicies } = await supabase
    .from('policies')
    .select('*, insurance_companies(*), road_services(*)')
    .eq('group_id', policy.group_id);
  
  // جلب كل دفعات الباقة
  const policyIds = packagePolicies.map(p => p.id);
  const { data: allPayments } = await supabase
    .from('policy_payments')
    .select('*')
    .in('policy_id', policyIds);
  
  // جلب كل ملفات الباقة
  const { data: allFiles } = await supabase
    .from('media_files')
    .select('*')
    .in('entity_id', policyIds)
    .eq('entity_type', 'policy');
}
```

## الخلاصة

**الفكرة الأساسية**: 
- الباقة = كيان واحد موحد
- كل شيء (الدفعات + الملفات) يُعرض مجمّع
- جدول يوضح مكونات الباقة (الأنواع + الأسعار + التواريخ)
- لا حاجة للتنقل بين وثائق الباقة المنفصلة

**الفوائد**:
- ✅ رؤية مالية واضحة للباقة ككل
- ✅ لا ارتباك حول أين الدفعات
- ✅ ملفات مجمّعة في مكان واحد
- ✅ تجربة مستخدم أبسط
