
# خطة تحسين بطاقات الوثائق في صفحة الوثائق

## ملخص المتطلبات
بناءً على الصور المرجعية والمتطلبات:
1. **تصغير حجم البطاقات** - لإظهار المزيد من الوثائق على الشاشة
2. **إظهار تفاصيل الباقة مباشرة** - بدون استخدام Accordion (التوسيع/الإغلاق)
3. **إضافة معلومات الإنشاء** - من أنشأ الوثيقة ومتى (تاريخ الإنشاء)
4. **تحسين الفهم للمدير** - عرض أوضح للمعلومات

---

## التغييرات المطلوبة

### 1. إعادة تصميم البطاقة

#### التصميم الحالي:
- بطاقات كبيرة مع padding = p-4
- تفاصيل الباقة داخل Accordion
- لا يوجد عرض لتاريخ الإنشاء ومن أنشأها

#### التصميم الجديد (مثل PolicyTreeView):
```text
┌──────────────────────────────────────────────────────────────────────────┐
│ [باقة] [ثالث] + [إلزامي] + [خدمات طريق]  [سارية] [✓ مدفوع]    ₪2,300   │
│                                                                          │
│ 🏢 الشركة        🚗 السيارة     📅 الفترة           👤 العميل          │
│ اراضي مقدسة      12345678      02/02 → 02/02/27     رأفت الجولاني      │
│                                                                          │
│ ── مكونات الباقة ──────────────────────────────────────────────────────│
│ [ثالث]      اراضي مقدسة   02/02/26 → 02/02/27   ₪1,000   محمد 10/02    │
│ [إلزامي]    منورا         02/02/26 → 02/02/27   ₪1,000   محمد 10/02    │
│ [طريق]      شركة اكس      02/02/26 → 02/02/27   ₪300     محمد 10/02    │
│                                                   [📧] [👁] [⋮]         │
└──────────────────────────────────────────────────────────────────────────┘
```

### 2. التغييرات التقنية على `PolicyCardsView.tsx`

#### أ. تصغير الحجم
- تقليل padding من `p-4` إلى `p-3`
- تصغير حجم النص والأيقونات
- تقليل المسافات بين العناصر (`gap-2` بدلاً من `gap-3`)

#### ب. إزالة الـ Accordion
- إظهار مكونات الباقة دائماً (بدون توسيع/إغلاق)
- إزالة `expandedGroups` state
- عرض المكونات كجدول صغير مضمن

#### ج. إضافة معلومات الإنشاء
- إضافة `created_at` للـ query في `Policies.tsx`
- عرض اسم المنشئ وتاريخ الإنشاء لكل مكون في الباقة
- تنسيق: `اسم الموظف • 10/02/2026`

#### د. تبسيط التصميم
- عرض البيانات بشكل أفقي (row-based)
- استخدام ألوان أخف للخلفيات
- تقليل عدد الـ Badges

## التفاصيل التقنية

### 1. تحديث PolicyRecord interface:
```typescript

  // ... existing fields ...
  created_at?: string; // إضافة تاريخ الإنشاء
}
```

### 2. تحديث الـ query في Policies.tsx:
```typescript
.select(`
  *,
  created_at, // إضافة هذا الحقل
  clients(...),
  ...
`)
```

### 3. إعادة هيكلة البطاقة:
- إزالة `useState` للـ `expandedGroups`
- إزالة `toggleGroup` function
- عرض `allPolicies` مباشرة في البطاقة
- إضافة row لكل policy component مع:
  - نوع الوثيقة (Badge)
  - اسم الشركة
  - الفترة
  - السعر
  - من أنشأها + متى

### 4. تنسيق التاريخ:
```typescript
const formatCreatedAt = (dateStr: string | null) => {
  if (!dateStr) return '';
  return new Date(dateStr).toLocaleDateString('en-GB', {
    day: '2-digit',
    month: '2-digit',
    year: '2-digit'
  });
};
```
