
# إضافة زر اختبار الاتصال وحقل التحويلة للمستخدمين

## نظرة عامة
سأقوم بإضافة:
1. **زر اختبار الاتصال** في صفحة إعدادات IPPBX للتأكد من صحة الإعدادات
2. **حقل رقم التحويلة** في صفحة إدارة المستخدمين لتعيين تحويلة لكل موظف

---

## أين تجد Extension Password؟

بناءً على وثائق API التي أرسلتها، الـ **extension_password** هو:

```text
كلمة مرور التحويلة مشفرة بـ MD5
```

**للحصول عليها من MasterPBX:**

1. في لوحة التحكم MasterPBX → **Call Routing** (أو Extensions)
2. اختر التحويلة المطلوبة (مثل 101)
3. ابحث عن **SIP Password** أو **Extension Password**
4. هذه الكلمة يجب تشفيرها بـ MD5 قبل إدخالها

**مثال:**
- إذا كانت كلمة المرور: `123456`
- MD5 Hash: `e10adc3949ba59abbe56e057f20f883e`

**ملاحظة:** عادةً تكون كلمة مرور واحدة لجميع التحويلات، أو يمكنك إيجادها في API Settings.

## التعديلات المطلوبة

### 1. صفحة AuthSettings.tsx - إضافة زر اختبار الاتصال

سأضيف:
- حقل إدخال لرقم هاتف الاختبار
- حقل إدخال لرقم التحويلة للاختبار
- زر "اختبار الاتصال"
- عرض نتيجة الاختبار (نجاح/فشل)

### 2. صفحة AdminUsers.tsx - إضافة حقل رقم التحويلة

سأضيف عمود جديد في جدول المستخدمين النشطين:
- **رقم التحويلة (PBX Extension)** - حقل قابل للتعديل
- حفظ تلقائي عند تغيير القيمة

## التفاصيل التقنية

### A) زر اختبار الاتصال في AuthSettings

```text
┌─────────────────────────────────────────────────────────┐
│  اختبار الاتصال السريع                                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  رقم هاتف للاختبار: [_____________________]            │
│                                                         │
│  رقم التحويلة:      [_____________________]            │
│                                                         │
│  [🔊 اختبار الاتصال]                                    │
│                                                         │
│  ⚠️ سيتم الاتصال برقم الهاتف عبر التحويلة المحددة      │
└─────────────────────────────────────────────────────────┘
```

**المنطق:**
1. إدخال رقم هاتف ورقم تحويلة
2. استدعاء Edge Function `click2call`
3. عرض نتيجة الاتصال

### B) حقل التحويلة في AdminUsers

في جدول المستخدمين النشطين:
| الاسم | البريد | الفرع | الدور | التحويلة | الإجراءات |
|-------|--------|-------|-------|----------|-----------|
| أحمد  | ...    | ...   | موظف  | [101]    | ...       |

**عند التعديل:**
- تحديث `profiles.pbx_extension` مباشرة
- إظهار Toast بنجاح/فشل العملية

## مثال على Token ID و Extension Password

من الوثائق التي أرسلتها:
```json
{
    "token_id": "aBcD21RfdoNklssp",      // من API Settings في MasterPBX
    "phone_number": "052XXXXXXX",         // رقم العميل المراد الاتصال به
    "extension_number": "101",            // رقم تحويلة الموظف
    "extension_password": "e10adc...83e"  // MD5 hash لكلمة مرور التحويلة
}
```

**أين تجد Token ID:**
- MasterPBX → **System Config** → **API Settings** → **Token ID**
- أو في قسم **API Guide** في القائمة الجانبية
