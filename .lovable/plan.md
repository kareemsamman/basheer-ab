
# تحويل صفحة الوثائق من بطاقات إلى جدول موحد

## نظرة عامة

تحويل صفحة `/policies` من عرض البطاقات (Cards) إلى جدول احترافي موحد مناسب للمراجعة المالية والتدقيق. كل صف يمثل وثيقة واحدة (حزمة أو مفردة).

---

## مواصفات الجدول

### الأعمدة (من اليمين لليسار - RTL)

| العمود | المحتوى | ملاحظات |
|--------|---------|---------|
| **عدد الملفات** | رقم فقط (1, 3, 12) | للحزمة = عدد الوثائق الداخلية، للمفردة = 1 |
| **العميل** | اسم العميل + رقم الهاتف (نص ثانوي) | سطرين |
| **محتوى الوثيقة** | شرائح ملونة (إلزامي، شامل، خدمات طريق) | باستخدام Badge components |
| **رقم السيارة** | لوحة السيارة | نص عادي |
| **الفترة** | تاريخ البداية → النهاية | بتنسيق DD/MM/YYYY |
| **السعر** | سعر الوثيقة الفردية (₪) | |
| **الإجمالي** | المجموع للحزمة | خط عريض/بارز |
| **أنشأها** | اسم الموظف | |
| **حالة الوثيقة** | سارية / منتهية / ملغاة / محولة | شريحة ملونة |
| **حالة الدفع** | مدفوع / متبقي ₪X | شريحة ملونة |

---

## قواعد التفاعل

- **لا أزرار إجراءات** - لا قائمة kebab، لا أيقونات حذف/تعديل
- **الصف بالكامل قابل للنقر** → يفتح صفحة تفاصيل الوثيقة
- **مؤشر الفأرة** يتغير لـ `cursor-pointer`
- واجهة نظيفة بدون تشتيت

---

## السلوك البصري

- صفوف متناوبة (Zebra striping) أو فواصل خفيفة
- شرائح الحالة متوافقة مع الألوان الموجودة:
  - سارية → أخضر
  - منتهية → رمادي
  - ملغاة → أحمر
  - محولة → برتقالي
- الإجماليات بخط عريض
- أسماء طويلة مقطوعة مع tooltip

---

## الحالات الخاصة

| الحالة | المعالجة |
|--------|----------|
| حزمة بحالات دفع مختلطة | عرض المتبقي الإجمالي |
| منتهية لكن غير مدفوعة | حالة الوثيقة ≠ حالة الدفع (منفصلتان) |
| أسماء طويلة | قص مع `truncate` و tooltip |

## التفاصيل التقنية

### الملفات المتأثرة

| الملف | التغيير |
|-------|---------|
| `src/components/policies/PolicyTableView.tsx` | **ملف جديد** - مكون الجدول الرئيسي |
| `src/pages/Policies.tsx` | استبدال `PolicyCardsView` بـ `PolicyTableView` + إضافة `phone_number` للـ query |
| `src/components/policies/cards/types.ts` | إضافة `phone_number` لـ interface |

### هيكل البيانات

استخدام نفس منطق `PolicyCardsView` للتجميع:
- `groupedPolicies` - تجميع الوثائق حسب `group_id`
- `getPackagePaymentStatus()` - حساب حالة الدفع للحزمة
- `getPolicyStatus()` - تحديد حالة الوثيقة

### تحديث Query لإضافة رقم الهاتف

```typescript
// Before
clients(id, full_name, less_than_24, under24_type, ...)

// After  
clients(id, full_name, phone_number, less_than_24, under24_type, ...)
```

### أنماط CSS المستخدمة

- `cursor-pointer` على كل صف
- `hover:bg-muted/50` للتمييز عند التمرير
- `even:bg-muted/20` للصفوف المتناوبة
- `truncate` للنصوص الطويلة
- استخدام `Badge` من shadcn للشرائح
